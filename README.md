# competition-projects

Practical data science and AI projects based on competitions and
real-world problem solving.
This repository focuses on end-to-end analytical workflows,
from problem definition to final reporting.

공모전 및 실제 문제 해결을 기반으로 한 데이터 사이언스·AI 프로젝트 저장소입니다.  
문제 정의부터 분석, 모델링, 평가, 결과 정리까지 **전체 분석 흐름(end-to-end workflow)**을 기록하는 것을 목표로 합니다.

---

## Purpose of this repository

### English
The purpose of this repository is to demonstrate practical problem-solving skills,
not just model performance.
Each project emphasizes:
- understanding the problem context
- designing a clear analysis pipeline
- interpreting results and limitations
- communicating findings clearly

This repository prioritizes **clarity and reproducibility** over leaderboard scores.

### 한국어
이 저장소의 목적은 단순한 모델 성능이 아니라,
**실전 문제 해결 능력**을 보여주는 데 있습니다.

각 프로젝트는 다음을 중점적으로 다룹니다.
- 문제 맥락에 대한 이해
- 명확한 분석 파이프라인 설계
- 결과 및 한계에 대한 해석
- 분석 결과를 전달하는 능력

리더보드 점수보다 **분석의 명확성과 재현성**을 우선합니다.

## Standard project workflow

### English
Each project follows a consistent workflow:
1. Problem definition
2. Exploratory data analysis (EDA)
3. Feature engineering and preprocessing
4. Modeling and evaluation
5. Result interpretation and reporting

### 한국어
각 프로젝트는 다음과 같은 공통 흐름을 따릅니다.
1. 문제 정의
2. 탐색적 데이터 분석(EDA)
3. 특성 엔지니어링 및 전처리
4. 모델링 및 성능 평가
5. 결과 해석 및 보고서 작성

---

## Repository structure

### English
Each competition or project directory contains:
- `README.md` : problem description, approach, and result summary
- `notebooks/` : EDA and modeling experiments
- `src/` : reusable scripts (preprocessing, features, training)
- `results/` : figures, tables, and evaluation outputs
- `data/` : excluded from git (only links or instructions provided)

### 한국어
각 공모전/프로젝트 폴더는 다음 구조를 따릅니다.
- `README.md` : 문제 설명, 접근 방법, 결과 요약
- `notebooks/` : EDA 및 모델 실험 노트북
- `src/` : 재사용 가능한 코드 (전처리, 특징 생성, 학습)
- `results/` : 그래프, 표, 성능 지표 정리
- `data/` : Git에 포함하지 않음 (데이터 링크 또는 준비 방법만 제공)

## Data policy

### English
- Raw datasets are not uploaded to this repository.
- Only publicly available datasets or dataset links are used.
- Data preparation steps are clearly documented.

### 한국어
- 원본 데이터는 저장소에 업로드하지 않습니다.
- 공개 데이터 또는 데이터 접근 링크만 사용합니다.
- 데이터 준비 과정은 문서로 명확히 정리합니다.

## Reporting principles

### English
- Clear explanation of assumptions and constraints
- Emphasis on interpretation, not just metrics
- Visual summaries for key findings

### 한국어

- 지표 수치보다 결과 해석에 집중
- 핵심 결과를 시각적으로 요약
